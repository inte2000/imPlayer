# 开发日志20060510



这一期的内容是继续完成上一次的任务，即跑通一个完整的外部解码器插件机制。这一期的主要任务有完成 libsnd_decoder 插件的全部功能，创建一个 CAudioDecoder 类的子类，用于封装插件动态库，完成从插件配置文件到 CDecoderFactory 中的解码器列表的最后一步。当然，最后就是播放一个新支持的媒体文件。

## 1 完成 libsnd_decoder

​        上一期完成的插件动态库项目是一个空的框架，所以先要给这个库引入真正的 libsndfile 库。这个库已经实现安装好了，只需通过 cmake 引入即可。然后是一些辅助函数，具体安排在 libsndfunc 文件中。还有就是提供一个使用 libsndfile 库操作媒体流的播放控制对象。

​        整个过程我拆分成三四个任务，有很多有意思的发现。比如第一个任务要求在引入 libsndfile 库的同时，给出一个将 libsndfile 库定义的文件格式转成 iplayer 在 audioinfo.h 中定义的文件格式常量，我要求它提供给一个转换函数。这里面有几个关键点，首先 ChatGPT 要识别 audioinfo.h 中定义的文件格式常量的命名模式，即 “StreamFormat+xxx”，其中 xxx 就是文件格式，一般体现为扩展名。其次是识别 libsndfile 库定义的格式常量，然后做映射转换。结果 ChatGPT 不仅正确给出这个转换函数，还提供了一个打开文件，读取库定义的格式，然后转换成 iplayer 格式常量的全链条封装，并且将其用在 Plus_ParseFileTypeID 导出函数中。这个导出函数的目的就是解析文件，获取文件类型，以便解码器能正确匹配。这让我大开眼界，因为我并没有在任务中提到这二者的关系，也没要它这么做。

​        在生成 LibSndPlayCtrl 类的时候，有一个难点就是按照 iplayer 的设计，解码器并不直接操作文件，取而代之的是一个代表数据流的 DataStream 抽象对象，这个流可以是文件流，也可以是内存数据块，也可以是网络流。libsndfile 库支持从抽象数据源中获取数据的方法，就是创建虚拟io对象。很显然，ChatGPT 也是不负众望，直接给出了我希望的代码，与此同时，还根据 Plus_xxx 系列导出函数的名字，自动关联了 LibSndPlayCtrl 类的成员，将 PlugMain 中的空函数也都填充好了，这让我必须对它高看一眼。

## 2 插件解码器抽象

​        为了对内置解码器和外部插件解码器统一管理，需要一个 CPluginDecoder 作为所有外置插件解码器的统一抽象，CPluginDecoder 也是 CAudioDecoder 的子类。为了让 ChatGPT 正确理解 CPlusinDecoder::AttachModule 成员函数的作用，我在任务描述中手写了一段代码，主要是让 ChatGPT 理解并使用闭包中捕捉的 dll（CDecoderDllWrapper 智能指针），不要再打开插件动态库重新加载（效率低）：

```cpp
DecoderCreator creator = [dll](uint32_t st) -> CAudioDecoder* {
    CPlusinDecoder* pDecoder = new CPlusinDecoder(st);
    if (pDecoder)
        pDecoder->AttachModule(dll);
    return pDecoder;        
};
```

结果它确实是理解了，AttachModule 成员函数的生成也是正确的，并且也是这么用的，但是人家是一行代码都没用我给出的例子，这是它的实现：

```cpp
DecoderCreator creator = [dll](uint32_t st) -> CAudioDecoder* {
    std::unique_ptr<CPluginDecoder> pDecoder = std::make_unique<CPluginDecoder>(st);
    pDecoder->AttachModule(dll);
    return pDecoder.release();
};
```

我觉得它有点看不起我，感觉自尊心有点被打击到了。

## 3 初始化 CDecoderFactory

​        接下来的一个任务就是将插件从配置到加载至 m_DecoderItems 的整个过程串起来。我原本嫌写任务描述麻烦，想手工写一个函数搞定，但是写了开头之后，想着这个项目本来就是为了验证 AI 的工作能力，于是就决定写个任务描述，让 ChatGPT 把函数补充完整。这是我写的半截代码：

```cpp
bool  InitDecoderFactory() 
{
    try
    {
        CDecoderFactory& factory = CDecoderFactory::GetInstance();
        factory.SetAppVersion(1, 0);
    }
    catch(const std::exception& e) 
    {
    
    }
}

// main 函数初始化部分：
if(!InitDecoderFactory())
{
    ...
    return -1;
}
```

我简单写了个任务描述：

```
main.cpp 中的 InitDecoderFactory 函数只完成了一部分，请完成剩余部分的工作，这部分工作主要是读取插件配置文件，初始化外部插件，并将其添加到 CDecoderFactory。可通过 GetPluginConfigFilePathname 获取插件配置文件的位置，利用 LoadPluginConfigFile 获取所有配置信息，然后通过 CreatePluginObjectByConfig 加载插件动态库，最后调用 CDecoderFactory::AddPluginObject 完成插件注册。
```

这是 ChatGPT 生成的结果：

```cpp
static void InitDecoderFactory()
{
    CDecoderFactory& factory = CDecoderFactory::GetInstance();
    factory.SetAppVersion(1, 0);

    const std::string pluginCfg = GetPluginConfigFilePathname();
    std::vector<PluginConfig> pluginItems;
    if (LoadPluginConfigFile(pluginCfg, pluginItems))
    {
        for (auto& plugCfg : pluginItems)
        {
            auto plugObj = CreatePluginObjectByConfig(plugCfg);
            if (!plugObj)
            {
                std::cerr << "Skip invalid decoder plugin: " << plugCfg.hostfile << std::endl;
                continue;
            }

            auto [ok, msg] = factory.AddPluginObject(*plugObj);
            if (!ok)
            {
                std::cerr << "Add decoder plugin failed: " << msg << std::endl;
            }
        }
    }

    const std::string decodercfg = GetDecoderConfigFilePathname();
    factory.LoadCustomDecoderConfig(decodercfg);
}
```

什么鬼，它把 try 给我扔了，返回值也改成 void 了，我有点不解。但是研究了一下代码，发现它是对的。因为 InitDecoderFactory() 函数中调用的函数都没有扔异常的，LoadPluginConfigFile() 函数用了 Json 库，这块有异常，但是这个函数内部 try 了。所以，我最初写的那个 try 实际上 try 了个寂寞。再说说返回值，加载外部解码器插件失败就直接结束程序有点欠考虑，外置解码器因为配置文件原因，失败很正常，不是还有内置解码器吗，咋能直接结束呢？所以 ChatGPT 把我的逻辑全改了，main 函数中也把错误返回的代码都删除了。让我觉得很抑郁的是，它的是对的，我觉得它更看不起我了，自尊心再次被打击了。

## 4 最后验证

​        最初内置解码器只有一个使用 mpg123 库的 mpeg audio 文件解码器，可以播放 mp3 文件，这个外置的解码器因为使用 libsndfile 库，支持的文件格式很多，flac，ogg 都支持。如果说程序一次就跑通了，你肯定不信，我也不信。出现了几个问题，比如 Plus_OnRegister 函数中没有注册文件类型和扩展名的关系，导致解码器不匹配。这个好改，给 ChatGPT 展示了映射关系，它自动补全了这个函数中缺失的注册代码。还有其他几个小问题，都是把错误原因提给 AI，它自己修改。
